# TODO Management

## Cleanup Rules

After completing work on a section of the TODO:

1. **Delete completed items** — Remove all tasks marked done from that section
2. **Delete empty sections** — Remove the section header if no tasks remain
3. **Keep only incomplete tasks** visible in the TODO

## What to Record as Done

Only record completion of changes that are **important for future development context** — items that affect Claude's understanding of how the project works or is structured.

### Include in Done
- New data structures or fields in the data flow
- Feature additions that change user-facing behavior
- User flow or navigation changes (new screens, new interactions)
- Architectural decisions or code organization changes

### Exclude from Done
- Cosmetic or layout tweaks (CSS, styling)
- Minor bug fixes without functional impact
- Internal refactoring that doesn't change behavior
- UI polish or visual adjustments

## Goal

Maintain a concise record of **functional and structural evolution** that helps Claude understand the project, not a complete mirror of every change.
